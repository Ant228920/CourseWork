# Система управління військовим округом

Повноцінна інформаційна система для управління військовим округом з усіма необхідними сутностями, CRUD операціями та 10 типами запитів згідно з вимогами.

## Технології

- **СУБД**: PostgreSQL 12+
- **Мова/GUI**: Python 3.8+ + Tkinter
- **Драйвер БД**: `psycopg2-binary`
- **Кодування**: UTF-8, всі назви колонок у БД — кирилицею (в лапках)

## Функціональність

### Основні сутності

1. **Військова ієрархія**:
   - Військові округи
   - Армії
   - Корпуси
   - Дивізії
   - Бригади
   - Військові частини
   - Роти, взводи, відділення

2. **Особовий склад**:
   - Військовослужбовці з різними званнями
   - Офіцерський склад (генерали, полковники, підполковники, майори, капітани, лейтенанти)
   - Рядовий та сержантський склад (старшини, сержанти, прапорщики, єфрейтори, рядові)
   - Військові спеціальності
   - Специфічні атрибути для різних категорій звань

3. **Техніка та озброєння**:
   - Бойова техніка (БМП, танки)
   - Транспортна техніка (тягачі, автотранспорт)
   - Спеціальна техніка (артилерія)
   - Стрілецька зброя (карабіни, автомати)
   - Артилерійське озброєння
   - Ракетне озброєння
   - Специфічні атрибути для кожного типу

4. **Інфраструктура**:
   - Місця дислокації
   - Споруди (казарми, штаби, склади, гаражі, медичні пункти, клуби, їдальні)

### CRUD операції

Система надає повний CRUD інтерфейс для всіх сутностей:
- **Create** - додавання нових записів
- **Read** - перегляд та пошук
- **Update** - редагування існуючих записів
- **Delete** - видалення записів

### 10 типів запитів

1. **Офіцерський склад** - загалом та по званнях, по частинах, дивізіях, арміях
2. **Рядовий та сержантський склад** - загалом та по званнях, по частинах, дивізіях, арміях
3. **Військова техніка** - загалом та по категоріях, по частинах, арміях, дивізіях, корпусах
4. **Озброєння** - загалом та по категоріях, по частинах, арміях, дивізіях, корпусах
5. **Найбільше/найменше частин** - армії, дивізії, корпуси з найбільшою/найменшою кількістю частин
6. **Перелік частин з керівниками** - всі частини з вказанням керівників
7. **Місця дислокації** - перелік місць дислокації по частинах, арміях, дивізіях, корпусах
8. **Частини з вказаною кількістю техніки/озброєння** - пошук частин за кількістю техніки або озброєння
9. **Військовослужбовці вказаної спеціальності** - по округах, арміях, частинах, підрозділах
10. **Споруди** - споруди частин, споруди з багатьма частинами, споруди без частин

## Встановлення та запуск

### 1. Встановлення PostgreSQL

Встановіть PostgreSQL 12+ та створіть базу даних:

```sql
CREATE DATABASE military_district_db WITH ENCODING 'UTF8' LC_COLLATE 'uk_UA.UTF-8' LC_CTYPE 'uk_UA.UTF-8';
```

### 2. Встановлення залежностей

```bash
python -m venv .venv
.venv\Scripts\activate  # Windows
# або
source .venv/bin/activate  # Linux/Mac

pip install -r requirements.txt
```

### 3. Налаштування підключення до БД

Створіть файл `.env` в корені проекту:

```env
DB_HOST=localhost
DB_PORT=5432
DB_NAME=military_district_db
DB_USER=postgres
DB_PASSWORD=your_password
```

### 4. Ініціалізація бази даних

```bash
# Створення схеми та таблиць
psql -U postgres -d military_district_db -f sql/full_setup.sql


# Додавання валідації та обмежень
psql -U postgres -d military_district_db -f sql/validation.sql
```

### 5. Запуск застосунку

```bash
.venv\Scripts\python app\app.py
```

## Ролі користувачів

- **Адміністратор** - повний доступ, управління користувачами
- **Оператор** - CRUD операції, запити, пошук
- **Авторизований** - перегляд, пошук, виконання запитів
- **Гість** - тільки перегляд

## Тестові дані

Система включає повний набір тестових даних:
- 3 військові округи
- 4 армії з корпусами та дивізіями
- 13 військових частин
- Повна ієрархія підрозділів (роти, взводи, відділення)
- 18 звань (офіцерські, сержантські, рядові)
- 10 військових спеціальностей
- 5 місць дислокації
- 20 військовослужбовців з різними званнями
- 10 одиниць техніки різних типів
- 10 одиниць озброєння різних типів
- 8 споруд

## Валідація даних

Система включає комплексну валідацію:
- Перевірка форматів інвентарних та серійних номерів
- Валідація дат народження та прийняття на службу
- Перевірка ієрархії командування
- Валідація призначення техніки та озброєння
- Перевірка унікальності та обов'язкових полів

## Особливості

- **Повна підтримка української мови** - всі назви таблиць та колонок українською
- **Гнучка ієрархія** - підтримка як дивізій, так і бригад
- **Детальна структура підрозділів** - роти, взводи, відділення
- **Специфічні атрибути** - окремі таблиці для різних типів техніки та озброєння

- **Швидкий пошук** - індекси для оптимізації запитів
